# stockcheck 서비스 개요

## 한 줄 정의

> "그래서 사? 말아?" — FOMO에 눈먼 초보를 위한 초간편 주식 판결 & 손실방어 서비스

종목 추천이나 수익 예측 앱이 **아님**.
"지금 들어가도 되는지" AI가 판결을 내려주고, 손실/수익을 실생활로 치환해서 보여주는 서비스.

---

## 타겟 사용자

- 2~30대 주식 왕초보
- 숫자가 공포이고, PER/PBR 같은 전문 용어가 외계어인 사람
- "그래서 지금 사? 말아?" 이 질문 하나만 머릿속에 있는 사람
- FOMO에 눈 돌아가서 충동 매수 직전인 사람

---

## 핵심 가치

| 기존 서비스 | stockcheck |
|------------|-----------|
| 복잡한 차트·지표 | "사지 마!" / "괜찮아 보여" 한마디 |
| 전문 용어 나열 | "치킨 5마리 날릴 확률 85%" 실생활 치환 |
| 숫자 공포 | 비유로 설명, 용어는 info 아이콘 뒤에 숨김 |
| 판단은 유저 몫 | AI가 판결 내려줌 (4단계) |

---

## 서비스 플로우

```
[로그인] → [종목 검색] → [수량 입력] → [AI 판결] → [시뮬레이션]
```

### 1단계 — 로그인
- 카카오 OAuth / Google OAuth
- 히스토리 저장을 위해 로그인 필수

### 2단계 — 종목 검색
- "오늘은 어떤 종목에 물리고 싶어?" 타이틀
- 종목명 또는 6자리 코드 입력
- 인기종목 칩 마퀴 (30초 순환) — 탭하면 바로 검색
- 실시간 이슈 배너 — issue-feed Edge Function (2시간 캐시), 6초 순환
- **미국 주식 자동 차단**: 영문 티커(AAPL 등) 또는 한글 미국 기업명(테슬라 등) 감지 시 "준비중" 바텀시트 노출 (API 호출 없음)

### 3단계 — 투자 수량 입력 (풀페이지)
- "몇 주 살 생각이야?"
- **주(株) 단위** 입력 (시스템 키보드, 최대 9999주)
- 현재가 카드 자동 표시 + 15초 주기 갱신
- 프리셋 칩: +1주 / +5주 / +10주 / +50주 (누적 방식)
- 예상 금액 실시간 표시 (주수 × 현재가)

### 4단계 — AI 판결

Edge Function이 4개 소스 **병렬** 수집 → 룰 기반 예비판정 계산 → **Claude Haiku가 직접 판결**

**데이터 수집 (병렬):**

| 소스 | 내용 |
|------|------|
| KIS API | 현재가, 52주 고/저, 일별 데이터, PER, 거래량 |
| Yahoo Finance + Alternative.me | VIX, 공포탐욕지수, KOSPI/KOSDAQ, DXY |
| 네이버 뉴스 API | 뉴스 제목/본문 실제 내용 (키워드 카운팅이 아닌 내용 전달) |
| DART API | 관리종목/투자주의/상장폐지 공시 2년치 |

**판결 흐름:**
1. 룰 기반 스코어링 (100점, 높을수록 위험)으로 예비판정 먼저 계산
2. 예비판정 + 전체 데이터 + 뉴스 내용 + DART 공시를 LLM에 전달
3. Claude Haiku가 모든 맥락 보고 최종 판결 등급 직접 결정
4. **강제 ban** (LLM 판정 무관): DART 관리종목/투자주의/상장폐지 감지 OR VIX ≥ 35

**룰 기반 스코어링 (예비판정 앵커용):**

| 축 | 배점 | 주요 지표 |
|----|------|---------|
| 종목 | 40점 | 52주 고점비율, PER, 거래량 배수, RSI |
| 시장 | 40점 | VIX, 공포탐욕, 코스피/코스닥 위치, Fed 금리, DXY |
| 뉴스 | 20점 | 루머 키워드 비율, 선반영 여부 |

**판결 4단계:**

| 등급 | 이모지 | 타이틀 |
|------|--------|--------|
| 절대금지 | 🤬 | 호구 입장 1초 전 |
| 대기 | 😤 | 호구 대기표 뽑는 중 |
| 괜찮아 보여 | 😎 | 인정. 가즈아! |
| 관망 | 🫠 | 이건 어렵군.. (데이터 완전 없을 때만) |

**결과 화면:**
- 판결 이모지 + headlineMent (10~20자 반말)
- 이슈 태그 바 (impactTag + priceSignalTag)
- 손실/수익 치환 (LOSS_LABELS/GAIN_LABELS 룩업, "날린다" 표현 금지)
- 이유 카드 4개 (price/value/volume/market — 비유 설명)
- "그래도 사면?" → 시뮬레이션

### 5단계 — 시뮬레이션
"그래도 사면?" 클릭 시 단기 + 장기 예측 표시:

- **단기**: 3일 기댓값 + best/worst 등락률 범위
- **장기**: 3개월 / 6개월 / 1년 — 기댓값 + best/worst 금액 밴드 (±1.28σ)
- 각 기간: 수익 금액 + GAIN_LABELS 치환 텍스트 + 이모지
- 일일 변동성 + 과거 연평균 수익률 기반 추정

## 디자인 컨셉

- **기본**: 토스 플랫 (TDS 기반 디자인 시스템)
- **판결 영역**: iOS 글래스모피즘 (backdrop-filter blur)
- **스티커 이모지**: 흰 테두리 + 그림자 + float 애니메이션 (CSS만, 이미지 불필요)
- **이모지 색 블리딩**: 이모지 자체 색이 글래스 배경에 연하게 번짐
- 참고: JOMO 앱 스타일

## 기술 스택

| 구분 | 기술 |
|------|------|
| 프론트엔드 | Vite + React 19 + Tailwind CSS v4 |
| 인증 | Supabase Auth (카카오/구글 OAuth) |
| DB | Supabase PostgreSQL |
| 서버 로직 | Supabase Edge Function (Deno) |
| 주식 데이터 | KIS (한국투자증권) Open API |
| 시장 데이터 | Yahoo Finance + Alternative.me (공포탐욕지수) |
| 공시 | DART API (opendart.fss.or.kr) |
| 뉴스 | 네이버 뉴스 검색 API |
| AI 분석 | Claude Haiku (`claude-haiku-4-5-20251001`) — Anthropic API |
| 배포 | Vercel (프론트) + Supabase (Edge Function) |

## DB 스키마

```
check_history — 판결 이력 (종목, 결과 JSON, 투자금, 판결 등급/점수)
issue_feed    — 실시간 이슈 피드 (2시간 캐시, KIS 거래량 상위 + 뉴스 + Claude Haiku)
```

RLS 활성화 — `check_history`는 본인 데이터만, `issue_feed`는 로그인 유저 전체 읽기.
