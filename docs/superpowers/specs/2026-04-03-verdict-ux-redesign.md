# stockcheck UX 개선 설계 문서

**날짜**: 2026-04-03 (최종 업데이트)
**범위**: SearchScreen 카피 + VerdictScreen 전면 개편 + 시뮬레이션 스토리 중심 전환
**타겟**: 주식 지식 제로인 2-30대 왕초보

---

## 1. 변경 배경

기존 화면이 금융 용어(PER, 52주 고점, VIX 등)를 그대로 노출해 타겟 유저가 이해하기 어려움.
판결 등급 라벨("괜찮아 보여", "대기")이 힘도 없고 재미도 없어 서비스 컨셉과 불일치.
이유 카드가 접힘 처리되어 핵심 정보가 숨겨짐.
LLM 문구가 금융 보고서 톤이라 재미없음.

## 2. SearchScreen 카피 변경

| 항목 | 기존 | 변경 |
|---|---|---|
| 메인 타이틀 | 어떤 주식에 눈 돌아갔어? | 오늘은 어떤 종목에 물리고 싶어? |
| 서브타이틀 | 사기 전에 AI한테 먼저 물어봐 | 물리기 전에 나한테 먼저 물어봐 |
| 마퀴 라벨 | 🔥 요즘 핫한 종목 | 🔥 지금 뜨고있는 종목 |

## 3. VerdictScreen 판결 등급 개편

### 3-1. 이모지 + 타이틀 (배지 제거, 크게 표시)

| 등급 | 기존 배지 | 새 이모지 | 새 타이틀 |
|---|---|---|---|
| ban | 절대금지 🚨 | 🤬 | 호구 입장 1초 전 |
| wait | 대기 🤔 | 😤 | 호구 대기표 뽑는 중 |
| ok | 괜찮아 보여 👀 | 😎 | 인정. 가즈아! |
| hold | 관망 🫥 | 🫠 | 이건 어렵군.. |

### 3-2. 글래스카드 내부 레이아웃

```
[글래스카드 — 등급색 틴트 유지]
  이모지 (64px)
  타이틀 (28px bold)  ← "호구 입장 1초 전"
  AI 멘트 (14px)      ← headlineMent (짧고 강하게, 이슈 반영)
  손실 치환            ← snarky 톤
```

기존 배지(label) 완전 제거. 기존 섹션 B(손실 치환 독립 카드) 제거 → 카드 안으로 이동.

### 3-3. 이슈 태그 바 (NEW)

글래스카드 바로 아래, 이유 카드들 위에 위치.

```
[글래스카드]

  💣 임상 실패   🔴 바이오 부정   📉 3일 하락
  (가로 스크롤, 넘치면 밀림)

[이유 카드들]
```

**태그 유형 & 색상:**

| 유형 | 색상 | 예시 |
|---|---|---|
| 이슈 유형 | 주황 틴트 | 💣 임상 실패 / 🏆 수주 확정 / 📊 실적 발표 |
| 섹터 영향 부정 | 빨강 틴트 | 🔴 바이오 부정 |
| 섹터 영향 긍정 | 파랑 틴트 | 🔵 방산 긍정 |
| 가격 신호 | 회색 틴트 | 📉 3일 연속 하락 / 📈 거래량 급등 |

**미노출 조건**: 이슈 데이터 없는 경우 (hold 등급 등) 태그 바 전체 미노출.

**LLM이 생성하는 필드 (reasons 응답에 추가):**
```json
{
  "issueType": "임상실패 | 수주 | 실적 | 거시경제 | 루머 | 기타",
  "sectorImpact": "긍정 | 부정 | 중립",
  "impactTag": "바이오 부정",
  "priceSignalTag": "3일 연속 하락"
}
```

## 4. 이유 카드 전면 개편

### 원칙
- **전문용어 완전 제거**: PER, 52주, VIX, RSI 등 화면에 절대 노출 금지
- **완전 펼침**: 접힘(toggle) 제거
- **동적 이모지**: 결과값에 따라 변경
- **진행바 없음**: 이모지 + 닉네임으로 충분
- **데이터 없는 카드**: 미노출 (카드 수 가변)

### 카드 구조

```
[이모지]  카드 제목 (일상어)
          닉네임 (bold)
          설명 (2줄, 완전 쉬운 언어)
```

### 카드 1 — 가격 위치 (건물층 메타포)

| 범위 | 이모지 | 닉네임 | 설명 |
|---|---|---|---|
| 85~100% | 🗼 | 꼭대기야 | 지금 거의 제일 비쌀 때야. 더 오를 공간이 없어. |
| 65~84% | 🏢 | 고층이야 | 꽤 비싼 편이야. 들어가기 애매해. |
| 40~64% | 🪜 | 딱 중간층이야 | 평균적인 가격이야. 나쁘지 않아. |
| 20~39% | 🏚️ | 폐가 수준이야 | 많이 내려왔어. 꽤 저렴해진 거야. |
| 0~19% | ⛺ | 텐트야 | 요즘 제일 쌀 때야. 완전 바닥 근처. |

### 카드 2 — 기업 가치

| 범위 | 이모지 | 닉네임 | 설명 |
|---|---|---|---|
| PER 60+ | 🤑 | 바가지 씌우는 중 | 이 회사가 돈 버는 속도로 따지면 투자금 돌려받는 데 60년 넘어. |
| PER 30~59 | 😒 | 좀 비싸 | 투자금 돌려받는 데 30~60년. 비싼 편이야. |
| PER 15~29 | 👌 | 적당해 | 회사 가치 대비 적당한 가격이야. |
| PER 1~14 | 💎 | 진짜 싸게 파는 중 | 실제 가치보다 훨씬 싸게 팔리고 있어. |
| 적자 | 💸 | 돈도 못 버는 회사 | 지금 수익이 없어. 가격 기준 자체가 없는 거야. |

### 카드 3 — 거래 분위기

| 범위 | 이모지 | 닉네임 | 설명 |
|---|---|---|---|
| 3배+ | 🔥 | 뭔가 터졌나봐 | 오늘 평소보다 훨씬 많이 사고 팔고 있어. 보통 이러면 무슨 소식이 있는 거야. |
| 1.5~3배 | 👀 | 좀 활발해 | 평소보다 활발해. 관심 받고 있는 거야. |
| 0~1.5배 | 😴 | 조용해 | 오늘 별 움직임 없어. |

### 카드 4 — 시장 기분

| 범위 | 이모지 | 닉네임 | 설명 |
|---|---|---|---|
| 공포 (0~30) | 😱 | 다들 겁쟁이 모드 | 요즘 투자자들이 겁먹고 팔고 있어. 이럴 때 혼자 사면 더 위험해. |
| 중립 (31~69) | 😐 | 평범해 | 시장 분위기 특별한 거 없어. |
| 탐욕 (70~100) | 🤪 | 다들 흥분 모드 | 다들 사고 싶어서 난리야. 이럴 때가 오히려 위험해. |

### 카드 5 — 뉴스 (있을 때만 노출)

LLM이 뉴스 헤드라인 기반으로 일상어 생성. 이슈 유형별 이모지 동적 적용.

| 이슈 유형 | 이모지 | 닉네임 예시 |
|---|---|---|
| 임상실패 | 💣 | 약발 안 먹힘 |
| 수주 | 🏆 | 대박 계약 터짐 |
| 실적 | 📊 | 실적 발표 앞뒤야 |
| 루머 | 🗣️ | 찌라시 돌고 있어 |
| 거시경제 | 🌍 | 시장 전체 영향 |
| 조용함 | 🦗 | 별 얘기 없어 |

## 5. LLM 프롬프트 전면 개선

### 페르소나 (30대 MZ 직장인 고인물)

```
너는 30대 직장인이야.
낮에는 회사 다니고 밤에는 토스·카카오페이로 주식 보는 MZ 고인물.
HLB 반토막, 카카오 존버 3년, 공모주 청약 줄섰다 꽝까지 다 경험했어.
유튜브 주식 채널 즐겨찾기만 20개고, 주식 갤러리도 매일 들어가.
지금 후배가 "이거 사도 돼?" 하고 카톡 보내왔어.
말리고 싶은 마음 반, 어차피 살 거 아는 마음 반.
직장인 특유의 자조적 유머로 현실을 직격해.
```

### 판결별 말투

```
🤬 절대금지 — 형이 직접 당한 것처럼 진심으로 말려
"아 이거 나 작년에 샀다가 반토막 났어. 진짜로."
"사지 마. 사면 나중에 나한테 욕해도 됨."
"회사에서 힘들게 번 돈인데 왜 여기다 넣으려고."

😤 대기 — 선배 톤으로 탐욕 참으라고
"3일만 참아. 딱 3일만."
"왜 지금 들어가려고 해. 이번 달 배민비 날리고 싶어?"
"지금 들어가면 왜 이때 샀지 하면서 잠 못 잠."

😎 인정 — 쿨하게, 절대 몰빵 금지
"나쁘진 않네. 근데 월급 전부는 넣지 마."
"타이밍은 맞는 것 같아. -10% 되면 버틸 수 있어?"

🫠 관망 — 솔직하게
"이건 나도 모르겠다. 진짜로."
"이럴 땐 그냥 안 사는 게 답임."
```

### 손실 치환 — 30대 MZ 소비 기준

```
1~3만원:   스벅 아아 10잔 / 마라탕 2번 / 편의점 야식 1주일 / 다이소 한 번 털기
3~10만원:  배민 2주치 / 올리브영 장보기 / 넷플릭스 4개월 / 콘서트 티켓 한 장
10~30만원: 나이키 운동화 / 닌텐도 게임 5개 / 쿠팡 로켓와우 2년치
30~70만원: 갤럭시 버즈 / 제주도 왕복 / 무신사 블랙프라이데이
70~150만원: 아이폰 구형 / 에어팟 프로 / 다이슨 드라이어
150만원+:  맥북 에어 / 유럽여행 비행기값

특수 치환 (임팩트용):
"당근마켓에서 팔려던 중고 아이폰값 날림"
"이번 달 카드값 갚을 돈"
```

## 6. 시뮬레이션 스토리 중심 전환 ✅ 구현 완료 (2026-04-03)

### 변경 방향

기존: 낙관적 장기 투자 수익 예측 (5/10/20년)
변경: **단일 기댓값 예측** + 범위 밴드 차트 (3개월/6개월/1년)

> 처음엔 best/worst 양방향으로 설계했으나, "너가 예측해서 하나만 알려줘야지"로 확정.
> 라벨은 총액 기준 아닌 **수익/손실 금액 기준** (예: "치킨 4마리 벌 수 있어")

### 확정된 구조

```
[헤더] 그래도 샀다면? 🤷

[범위 밴드 차트]
- 실선: 기댓값 (단일 예측)
- 음영: ±1.28σ 범위 밴드
- X축: 현재 / 3개월 / 6개월 / 1년
- 수익이면 파랑, 손실이면 빨강

┌──────────────┐
│ 📅 3개월 뒤  │
│ ☕ 아메리카노 4잔 벌 수 있어  │ ← 수익 시 파랑
└──────────────┘

┌──────────────┐
│ 📅 6개월 뒤  │
│ 🍗 치킨 2마리 벌 수 있어  │
└──────────────┘
```

### 계산 로직 (구현 완료)

- `calcScenario(months)`: 기댓값 = `annualReturn * years`
- 범위: `±annualVol * √years * 1.28` (±1.28σ)
- 라벨: `getRelatableUnit(|gain|)` 기반 → 기간별 다른 금액 → 다른 라벨
- `isPositive`: gain >= 0이면 긍정 표현, 음수면 손실 표현

### Edge Function projection 스키마 (최종)

```typescript
projections: {
  month3: { amount, gain, bestAmount, worstAmount, label, emoji }
  month6: { amount, gain, bestAmount, worstAmount, label, emoji }
  year1:  { amount, gain, bestAmount, worstAmount, label, emoji }
}
```

## 8. Phase 5 — 메인 이슈 피드 (추후 구현)

### 개요

토스증권 AI 분석 기능에서 차용. 단, 토스와 달리 "투자 기회"가 아닌 **"FOMO 차단 훅"** 으로 활용.

```
토스: "임상 실패 → 바이오 기회?" (기회 탐색)
stockcheck: "임상 실패 → 지금 사려고? 판결 먼저" (위험 경보)
```

### 화면 구조

마퀴(가로 스크롤 종목) 제거 → 이슈 카드로 교체.
카드 전체 탭 → 해당 종목 판결 시작 (버튼 없음).

```
[검색창]

💣 조심할 종목        ⚡ 지금 뜨는 이슈
┌──────────┐  ┌──────────┐
│ 한올바이오│  │ 한화오션 │
│ 임상 실패 │  │ 미국 수주 │
│ -11.7%   │  │ +7.7%    │
└──────────┘  └──────────┘
```

### 분석 로직

```
[Supabase Cron — 30분마다]
① KIS API: 거래량 상위 20종목 조회
   → 거래량 2배+ or 가격 ±3%+ 필터 → 이슈 후보 5~8개

② 네이버 뉴스 API: 각 종목 헤드라인 3개 수집

③ Claude Haiku:
   issueType / sectorImpact / oneLineSummary / plainExplain 생성

④ Supabase issue_feed 테이블 저장 (2시간 유효)

⑤ 메인 화면: issue_feed 최신 3~5개 표시
```

### DB 스키마

```sql
issue_feed (
  id uuid PK,
  stock_code text,
  stock_name text,
  price_change numeric,
  issue_type text,
  sentiment text,           -- 위험/주의/긍정
  emoji text,
  one_line text,
  plain_explain text,
  created_at timestamptz,
  expires_at timestamptz    -- 2시간 유효
)
```
